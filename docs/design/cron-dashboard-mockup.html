「执行细节」(run-detail-sheet) 关键设计

  1. 顶部状态药丸 + 时间 — 一眼看出这次成功/失败/运行中，不是一堆 dot 让人猜。
  2. 4 块 stats 横条 — 耗时 / Tokens / 工具调用次数 / Session ID。Session ID 可点击跳到完整对话。
  3. Tab 分流不同场景：
    • 对话（默认）— 把 stream-json 还原成时间线（user → assistant → tool calls）
    • 工具调用 — 单看 7 次 bash/read/edit，按耗时/状态过滤定位异常
    • 提示词 — 完整 prompt，方便调试
    • 原始日志 — stream-json 原文，给开发者排查
  4. 对话 timeline 元素 — 头像区分角色（user/Claude/tool），时间戳，tokens 增量，工具调用折叠成单行可展开（命令 + 状态药丸 + 耗时），失败的工具高亮红色。
  5. 失败定位的关键 — 对话流里直接看到哪一步 tool 红色，展开看 stderr，不用再翻日志。状态 3 例子：journalctl 超时，超时原因清晰可见。
  6. 运行中实时流 — 末尾光标闪烁，footer 显示「实时流式中」+ 自动跟随，可中止；停止流后变常规 detail。
  7. 上/下次 run 翻页 — sheet 底部翻页按钮，不用关 sheet 再点列表行。
  8. 右滑覆盖而非下方展开 — 不挤压历史列表，关闭后无错位。

Mobile 适配

  1. 三栏布局塌缩成 stack 导航 — 桌面项目栏 + 列表 + 详情并列；移动端「列表 → 任务详情 → run 细节」一次只显示一屏，左滑/返回按钮回退。
  2. 触摸目标 ≥ 44px — cron 行 56px、历史行 54px、底部按钮 42px，FAB 主按钮 44px+，Apple HIG 标准。
  3. 列表行精简 — 桌面 cron 行右侧"成功率徽章"，移动端只保留必要：状态点 / 名称 / 调度+下次（一行截断）/ 徽章 / ›。
  4. KPI 2×2 网格 — 桌面 1×4 横排在 360px 屏会被压扁；移动端改 2×2 正方形 grid，主要 KPI（下次运行 + 上次结果）放上排吸引眼球。
  5. run-sheet 改 bottom-sheet — 桌面右滑覆盖，移动端从下弹出占 88vh，顶部有 drag handle 可下拉关闭（符合 iOS/Android 系统习惯）；后面背景 dim 不滚动。
  6. tabs 横向滚动 — 4 个 tab 在 320px 屏放不下时变 horizontal scroll，不挤压换行。
  7. 工具调用卡片折行 — 桌面单行 [icon][name][summary][status][dur]，移动端命令独占第二行，避免超长命令被切到只剩 git。
  8. 底部主按钮 sticky — "立即执行"占满主区，旁边是 ‖（暂停）和 🗑（删）小图标按钮，FAB-style；适配 safe-area-inset-bottom。
  9. 底 tab 栏 — 会话/Agent/定时/更多，符合 dashboard 整体导航；当前项 active 高亮。
  10. 翻页按钮置底 — sheet 底部 ‹ 上次 / ↻ 重跑 / 下次 ›，单手拇指可达；不阻塞内容滚动。
