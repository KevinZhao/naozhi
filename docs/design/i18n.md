## 0. 背景与动机

当前 naozhi 没有任何 i18n 基础设施：
- **110 处 Go 字面量**（90% 中文，10% 英文），分布在 `internal/dispatch/commands.go`(60) / `dispatch.go`(21) / `apierr.go`(9) / `cli/todo.go`(4) / `cron/scheduler.go`(4) 等
- **879 处 HTML 中文字符** + **245 处 JS 中文字符**（`dashboard.html` / `dashboard.js`）
- `config.yaml` 中的 `transcribe.language` 仅是 AWS Transcribe BCP-47，与 UI i18n 无关
- `dashboard.html` 上的 `<html lang="zh">` 硬编码

**业务触发点**：
1. 飞书/Slack/Discord 同 bot 会被中英混用用户共享，英文用户看到中文错误信息会困惑
2. `localizeAPIError` 已经在做"英文→中文"的本地化，但反过来（英文用户收到英文）没有通路
3. Dashboard 被英文运维使用时，侧栏、help、cron 面板全中文

**不做 i18n 的代价**：产品要进海外市场时需要大规模重构；短期看每加一个新命令/错误都要中文 + 英文双写，只是时间问题。

## 1. 目标与非目标

### 1.1 目标
- **G1**：所有用户可见文案外置到消息字典，源码不再出现硬编码（测试/日志/panic 消息不算用户可见）
- **G2**：同一 bot 实例能同时服务 `zh-CN` 和 `en-US` 用户，按每条消息的 user locale 动态返回
- **G3**：IM 平台 webhook 或通过启发式能推断用户语言偏好；推断不到时回退 config default；**用户可用 `/lang` 命令一次性锁定或清除**（一期）
- **G4**：Dashboard 前端支持手动切换语言，持久化到 **cookie `nz_lang`**；JS 侧可同步一份到 localStorage 作辅助存储
- **G5**：新增文案必须走消息字典；CI 拦截字典 key 缺失 / 占位符不对齐 / 硬编码 CJK 增量

### 1.2 非目标
- **NG1**：~~第一期不做 `/lang`~~（v3 起已改为一期实现，见 §3.1）
- **NG2**：不做 LLM prompt 本地化；但保留 `{{system_locale_hint}}` 占位槽作为二期 backlog
- **NG3**：不做 RTL 语言支持（ar/he）；仅 zh-Hans、en 两种；繁体 `zh-TW` 当前归并到 `zh-CN`（见 §7 开放问题 7 及触发阈值）
- **NG4**：`internal/cli/*` 里 Claude CLI 的原生错误文本不翻译
- **NG5**：第一期不做字典热更新；提供 `NAOZHI_DEV_LOCALE_DIR` + `-tags dev` 作为开发逃生舱

## 2. 总体架构

```
┌─────────────────────────────────────────────────────────────────┐
│ IM platforms (feishu / slack / discord / weixin)                │
│   webhook → resolve UserLocale → IncomingMessage.UserLocale     │
│     • feishu: CJK 启发式 + /lang override                       │
│     • slack:  users.info.locale，LocaleCache(team+user, 30m)    │
│     • discord: interaction.locale / member.locale 直接拿        │
│     • weixin: 恒空（API 不暴露）                                │
└────────────────────────────┬────────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────────┐
│ internal/i18n (NEW, 零 net/http 依赖)                           │
│   ├── bundle.go       embed.FS + Load() → 不可变 Bundle         │
│   ├── locale_cache.go LocaleCache 接口（供各 transport 复用）   │
│   ├── locale/{zh,en}.yaml                                       │
│   ├── resolver.go     ResolveDashboard / ResolveIM(IMInput)     │
│   ├── printer.go      Printer 持 *Bundle，T(key, args)          │

│   └── flatjs.go       预序列化 template.JS cache                │
└────────────────────────────┬────────────────────────────────────┘
                             │
┌──────────────┬─────────────┴─────────────┬──────────────────────┐
│ dispatch/    │ cron/ cli/todo/           │ server/ (Dashboard)  │
│ commands.go  │ scheduler.go              │ HTTP handlers        │
│ apierr.go    │                           │   cookie nz_lang     │
│ dispatch.go  │ T(sess.Locale, ...)       │   ?lang=en           │
│ /lang 处理   │   session.Locale 三档     │   Accept-Language    │
│ (auto 清除)  │   弱固化                  │   (q-value)          │
└──────────────┴───────────────────────────┴──────────────────────┘
                                                      │
                                       ┌──────────────▼──────────┐
                                       │ static/dashboard.html   │
                                       │   html/template 预渲染  │
                                       │   window.__t(key, args) │
                                       └─────────────────────────┘
```

**核心决策**：
- **单一事实源**：YAML 文件 `internal/i18n/locale/*.yaml`，`embed.FS` 编进二进制
- **Bundle 不可变**：构造后只读；`Printer` 持 `*Bundle` 指针（不持 map 引用），为未来 `atomic.Pointer` swap 预留通道（NNH1）
- **i18n 包零 HTTP 依赖**：`ResolveDashboard(cookie, query, acceptLang)` 纯字符串签名
- **IM Resolve API 结构体化**：`ResolveIM(IMResolveInput) → (locale, source)`，避免 5 string 顺序错（NNM4）
- **前端用 `window.__t(key, args)`**；后端用 `*i18n.Printer.T(...)` + AST 扫描
- **三档置信度**：`user` > `platform` > `heuristic` > `""`（新 session）；低置信可被高置信覆盖
- **`/lang` 一期实现**：`show / zh / en / auto`；`auto` 由 dispatcher **在调用 ResolveIM 前**显式清 session.LocaleSource（NNB1）
- **`LocaleCache` 接口在 i18n 包**：Slack/Discord 共用；Slack cache key = `team_id:user_id`（NNM2）
- **CSRF 防线**：写 cookie endpoint 要求 auth + Content-Type=json + body 校验；错误响应本地化（NNM3）

## 3. 关键设计

### 3.1 Locale 来源与优先级

#### Dashboard 链（4 跳）

```
?lang= query param (单次覆盖，不持久化)
         ↓
cookie nz_lang (用户在 Settings 设置过)
         ↓
Accept-Language header (q-value 排序 + x/text Matcher 匹配)
         ↓
config.i18n.default_locale (default: "zh-CN")
```

#### IM 链（三档置信度覆盖模型）

`session.LocaleSource` 三档：

| 档位 | 值 | 来源 | 覆盖规则 |
| --- | --- | --- | --- |
| 高 | `"user"` | `/lang zh` / `/lang en` / Dashboard 显式切换 | 不被任何来源覆盖；唯一清除通道是 `/lang auto` |
| 中 | `"platform"` | Slack `users.info.locale` / Discord `interaction.locale` | 被 `user` 覆盖；被新 `platform` 值覆盖 |
| 低 | `"heuristic"` | CJK 比例启发式 | 被 `platform` 或 `user` 覆盖 |
| 兜底 | `""` | 新 session / `/lang auto` 清除后 | 任何来源都可落位 |

**每条消息的解析（dispatcher 伪代码）**：

```go
// Step 1: /lang auto 命令的 dispatcher 层处理（NNB1）
// 必须在 ResolveIM 之前完成，因为 ResolveIM 不接受"清除"信号
if cmd == "/lang auto" {
    sess.Locale = ""
    sess.LocaleSource = ""
    sess.MarkDirty()
    return replyWith(p.T("dispatch.command.lang.cleared"))
}

// Step 2: /lang zh|en 命令
if cmd == "/lang zh" || cmd == "/lang en" {
    locale := normalizeLangArg(cmd)  // → "zh-CN" 或 "en-US"
    sess.Locale = locale
    sess.LocaleSource = "user"
    sess.MarkDirty()
    p := bundle.For(locale)
    return replyWith(p.T("dispatch.command.lang.updated", map[string]any{"locale": locale}))
}

// Step 3: 普通消息走 ResolveIM
locale, source := bundle.ResolveIM(i18n.IMResolveInput{
    PlatformHint: msg.UserLocale,
    PrevLocale:   sess.Locale,
    PrevSource:   sess.LocaleSource,
    MessageText:  msg.Text,
    // UserOverride 字段仅在 dispatcher 处理完 /lang 前有用；
    // /lang 命令已走 Step 1/2 短路，这里不传
})
if locale != sess.Locale || source != sess.LocaleSource {
    sess.Locale, sess.LocaleSource = locale, source
    sess.MarkDirty()
}
p := bundle.For(locale)
// ...
```

**`ResolveIM` 内部算法**（对应 §3.4 API）：

```
ResolveIM(in IMResolveInput) → (locale, source):
  // "user" 锁定优先：
  if in.PrevSource == "user" && in.PrevLocale != "":
    return (in.PrevLocale, "user")
  // platform 值（无论是否与 prev 相同都采纳新值，允许用户改 Slack 设置生效）：
  if in.PlatformHint != "":
    normalized := NormalizeLocale(in.PlatformHint)
    if normalized != "":
      return (normalized, "platform")
  // 沿用 prev platform（本条消息没有 platform hint 但之前有）：
  if in.PrevSource == "platform" && in.PrevLocale != "":
    return (in.PrevLocale, "platform")
  // 启发式：
  if h, confident := bundle.Heuristic(in.MessageText); confident:
    return (h, "heuristic")
  // 沿用 prev heuristic：
  if in.PrevSource == "heuristic" && in.PrevLocale != "":
    return (in.PrevLocale, "heuristic")
  // 兜底：
  return (bundle.defaultLocale, "")
```

**`/lang` 命令** key 空间 `dispatch.command.lang.*`：

```
/lang          → "当前语言: {locale} （来源: {source}）"
/lang zh       → 设 ("zh-CN", "user"), reply "语言已切换为 zh-CN"
/lang en       → 设 ("en-US", "user"), reply "语言已切换为 en-US"
/lang auto     → 清 ("", ""),          reply "已清除语言偏好，将自动识别"
/lang xxx      → reply "无效参数; 支持 zh / en / auto"
```

**`/help` 命令文案必须包含 `/lang` 子命令说明**（NNH3）—— PR3a 的 `/help` 迁移同时更新文案。

**IM 平台识别现状表**：

| 平台 | 来源 | 第一期实现 |
| --- | --- | --- |
| 飞书 | Webhook **不携带 user language**；`i18n_info` 是 outbound 卡片机制；`i18n_name` 是姓名多语版 | 启发式 + `/lang` override；不调用户信息 API |
| Slack | `events.user` → `users.info` → `user.locale`，需 `users:read` scope | `LocaleCache` 实现（见 §3.2）；cache key `team_id:user_id`；失败降级启发式 |
| Discord | interaction `locale` / DM `author.locale` | 直接拿；缺则启发式 |
| Weixin | 公众号 API 不暴露 | 恒走启发式 |

### 3.2 `LocaleCache` 接口（NNM2：移到 i18n 包以便复用）

```go
// internal/i18n/locale_cache.go
package i18n

// LocaleCache caches user_id → locale lookups per workspace.
// Implementations: slack (team_id scoped), discord (guild_id scoped if needed).
// Consumers pass a Scope string (e.g. team_id / guild_id) to prevent cross-
// workspace pollution (NNM2/NB2 修复).
type LocaleCache interface {
    // Get returns cached locale for (scope, userID). scope MUST be the platform
    // workspace identifier (Slack team_id, Discord guild_id). Single-workspace
    // deployments MAY pass "" as scope but must do so consistently.
    Get(scope, userID string) (locale string, ok bool)
    // Put records (scope, userID) → locale with default TTL.
    Put(scope, userID, locale string)
}

// NewLRU returns a size-bounded LRU implementation with TTL.
// size: max entries; ttl: entry lifetime.
func NewLRU(size int, ttl time.Duration) LocaleCache
```

**Slack transport 使用**（`internal/platform/slack/locale.go`）：

```go
func (s *Slack) fetchUserLocale(ctx context.Context, teamID, userID string) string {
    if loc, ok := s.localeCache.Get(teamID, userID); ok {
        return loc
    }
    // 单 workspace 部署时 teamID 可能为空，接口允许但必须一致（NNM6）
    info, err := s.client.UsersInfo(ctx, userID)
    if err != nil { return "" }
    loc := i18n.NormalizeLocale(info.Locale)
    if loc != "" {
        s.localeCache.Put(teamID, userID, loc)
    }
    return loc
}
```

**NNM6 降级**：若 Slack webhook 不提供 `team_id` 字段或 naozhi 配置 `slack.single_workspace=true`，transport 传空串 scope；LocaleCache 接口允许但调用方**必须一致**，否则污染。单测覆盖 "team_id 有" / "team_id 空" 两路径。

### 3.3 Locale 归一化与 Fallback 表

`NormalizeLocale(s string) string` 规则（按顺序）：

1. trim 空格；全转小写
2. `.UTF-8` / `.GBK` 等 POSIX 编码后缀 → 剥掉（`zh_cn.utf-8` → `zh_cn`）
3. `_` → `-`（`zh_cn` → `zh-cn`）
4. 前缀 `zh` / `zh-hans*` / `zh-hant*` / `zh-*` → `zh-CN`
5. 前缀 `en` / `en-*` → `en-US`
6. 不在白名单 → 返回空字符串 `""`

完整 fallback 表：

| 输入 | 归一化 | 白名单命中 |
| --- | --- | --- |
| `zh`, `zh-CN`, `zh_cn`, `zh-Hans`, `zh-Hans-CN`, `zh-SG` | `zh-CN` | ✓ |
| `zh-TW`, `zh-HK`, `zh-Hant`, `zh-Hant-TW`（**当前归并，见 §7 开放问题 7**）| `zh-CN` | ✓（降级）|
| `en`, `en-US`, `en-GB`, `en_us`, `Zh-cN`（大小写混合经步骤 1）| `en-US` / `zh-CN` | ✓ |
| `ja`, `ja-JP`, `ko-KR`, `fr-FR` 等 | 空串 → default | ✗ |
| `zh_CN.UTF-8` | `zh-CN` | ✓ |
| `*`, `*.*` 等奇异 Accept-Language | 空串 → default | ✗ |

**`zh-TW → zh-CN` 归并对繁体用户是降级体验**；见 §7 开放问题 7 的触发阈值；`zh.yaml` 的 `_meta.script: Hans` 字段已**由 `Bundle.scripts` 读取**（NNM1 修复）。

**x/text Matcher**：对 `Accept-Language` header 走 `language.ParseAcceptLanguage(s)` → `matcher.Match(tags...)` 支持 q-value；简单字符串走自写 `NormalizeLocale`。

### 3.4 `platform.IncomingMessage` 与 `session.Session` 扩字段

```go
// internal/platform/platform.go
type IncomingMessage struct {
    // ... existing fields
    UserLocale string // BCP-47 canonical; "" means transport could not determine
}

// internal/session/session.go
type Session struct {
    // ... existing
    Locale       string // BCP-47 canonical; 可被高置信度来源覆盖
    LocaleSource string // "user" | "platform" | "heuristic" | ""
}
```

**谁负责填 IncomingMessage.UserLocale**：
- Slack/Discord transport：解析 webhook / 查 users.info 时填；Slack 失败降空串
- Feishu/Weixin transport：**不填**（留空），由 dispatch 层启发式兜底

**懒补迁移**（NNH4 修复措辞）：
- `Session.Load` 时若 `Locale == ""`：内存里填 `config.default_locale`，`LocaleSource = ""`
- **这是透明的内存视图修正，不保证立即落盘**；允许每次 Load 重复发生
- 若下一条消息的 `ResolveIM` 产生了与内存状态不同的值 → `MarkDirty()` → 自然 Save 持久化
- 若 `ResolveIM` 恰好算出相同值 → 不 Save，下次仍走懒补（幂等）
- 不在启动时批量 migration（避免启动时间放大 + 并发 race）

**向前兼容**（NNM8）：新字段 `Locale` / `LocaleSource` 对老 session 文件 JSON 解码走 Go 默认"忽略未知字段"语义，天然兼容；老版本 naozhi 读新格式文件也会忽略这两字段继续运行。**不要**把 session store 改为 strict mode（未知字段报错）。

### 3.5 `internal/i18n/` 包 API

```go
package i18n
// 零 net/http 依赖，保持纯净；HTTP 请求由调用方解析后传字符串参数

// Bundle holds all locales. Immutable after Load. Concurrent T() is safe.
// No Reload() API in v1. Future Reload MUST use atomic.Pointer[state] swap;
// since Printer holds only *Bundle (not map refs), swap is sufficient to
// redirect future T() calls while keeping live Printer values valid (NNH1).
type Bundle struct {
    defaultLocale string
    supported     []string
    heuristicCfg  HeuristicCfg                          // NNM5
    scripts       map[string]string                     // NNM1: locale → _meta.script
    msgs          map[string]map[string]*compiledTemplate
    flatJSCache   map[string]template.JS                // NH4/NM2: 预序列化 + key 排序（NNM10）
    flatMapCache  map[string]map[string]string          // NNH5: 供测试/CI lint 取
}

// Load validates config, builds compiled templates, precomputes JSON (keys
// sorted for byte-stable output → ETag friendly, NNM10).
// Returns error if defaultLocale=="" || !contains(supported, defaultLocale) ||
// len(supported)==0 || any YAML fails parse (NL2).
// devDir: if non-empty AND build tag "dev" is active, loads from OS FS (NL3).
func Load(defaultLocale string, supported []string, cfg HeuristicCfg, devDir string) (*Bundle, error)

// NewForTest builds a Bundle from an in-memory map; bypasses YAML.
// Intended for unit tests only.
func NewForTest(messages map[string]map[string]string) *Bundle

// Printer is locale-bound. Holds *Bundle pointer (not map refs) so future
// Reload via atomic.Pointer can redirect lookups (NNH1).
type Printer struct {
    locale string
    bundle *Bundle
}

func (b *Bundle) For(locale string) *Printer

// T renders a key with named args. Unknown key → "[" + key + "]".
// Missing arg in map → placeholder "{name}" unchanged.
// Extra args ignored.
func (p *Printer) T(key string, args ...map[string]any) string

// ResolveDashboard walks cookie > query > Accept-Language > default.
// Pure string signature to keep i18n package zero-HTTP.
func (b *Bundle) ResolveDashboard(cookie, query, acceptLanguage string) string

// IMResolveInput collects all inputs for IM resolution. Struct form avoids
// 5-string positional errors (NNM4).
type IMResolveInput struct {
    PlatformHint string // transport-provided, e.g. Slack users.info.locale
    PrevLocale   string // session.Locale
    PrevSource   string // session.LocaleSource ("user"|"platform"|"heuristic"|"")
    MessageText  string // for CJK heuristic
    // UserOverride intentionally omitted: /lang commands short-circuit in
    // dispatcher before calling ResolveIM (see §3.1 Step 1/2).
}

// ResolveIM walks the priority chain per §3.1. Always returns a valid
// (locale, source); locale is guaranteed to be in supported_locales.
func (b *Bundle) ResolveIM(in IMResolveInput) (locale, source string)

// NormalizeLocale canonicalizes per §3.3. "" for non-whitelisted.
func NormalizeLocale(s string) string

// Heuristic infers zh-CN vs en-US by CJK rune count (uses utf8.RuneCountInString,
// not byte length, NM4). Returns ("", false) if cfg.Enabled==false OR text has
// fewer than cfg.MinRunes runes (NNM7).
func (b *Bundle) Heuristic(text string) (locale string, confident bool)

// FlatMessages returns the locale's full flat map. Returned map is owned by
// Bundle; callers MUST NOT mutate. Used by CI lint tests and FlatMessagesJS.
// NNH5: restored from v3 where it was accidentally removed.
func (b *Bundle) FlatMessages(locale string) map[string]string

// FlatMessagesJS returns precomputed template.JS blob (key-sorted JSON) of the
// locale's full flat map. O(1); safe to embed directly in <script>. (NH4+NNM10)
func (b *Bundle) FlatMessagesJS(locale string) template.JS

// Scripts returns locale → _meta.script (e.g. "Hans", "Hant"). Read from YAML
// at Load time. For future zh-TW separation (§7 开放问题 7). (NNM1)
func (b *Bundle) Scripts() map[string]string
```

**占位符模板**：

```yaml
# zh.yaml
_meta:
```

### 3.6 非 IM 触发场景的 locale 策略

| 场景 | locale 来源 | 说明 |
| --- | --- | --- |
| Cron 通知（job 属于某 session） | `session.Locale` | cron job 记录 owner session key；通知时 lookup 取 |
| Cron 通知（独立 job 无 session） | `config.i18n.default_locale` | 独立 cron 无用户上下文 |
| Scratch drawer 注入 prompt | **恒英文** | 给 claude 的 prompt；英文稳定性最佳；用户可见区域走 session.Locale |
| Dashboard API 4xx/5xx | `ResolveDashboard(cookie, query, acceptLang)` | 错误响应的 body 文案走 `api.*` key 空间（NNM3）|
| Dispatcher panic 自动回复 | `msg.UserLocale` 已解析 | panic notify 走已有 IncomingMessage |
| Session 超时 / shim 重连通知 | `session.Locale` | session 已知 |
| 系统启动/关闭日志 | **不走 i18n** | 恒英文，给运维看 |

### 3.8 Dashboard 前端接入

**HTML 模板**（`dashboard.html.tmpl`，用 `html/template`）：

```go
// internal/server/dashboard_index.go
var dashboardTmpl = template.Must(
    template.New("dashboard.html").
        ParseFS(staticFS, "static/dashboard.html.tmpl"),
)

func (s *Server) renderDashboardHTML(w http.ResponseWriter, r *http.Request) {
    cookieVal := ""
    if c, _ := r.Cookie("nz_lang"); c != nil { cookieVal = c.Value }
    locale := s.i18n.ResolveDashboard(
        cookieVal,
        r.URL.Query().Get("lang"),
        r.Header.Get("Accept-Language"),
    )
    data := struct {
        Locale   string
        Messages template.JS // 预序列化缓存，O(1)（NH4）
    }{locale, s.i18n.FlatMessagesJS(locale)}
    w.Header().Set("Content-Type", "text/html; charset=utf-8")
    dashboardTmpl.Execute(w, data)
}
```

```html
<!-- dashboard.html.tmpl -->
<!DOCTYPE html>
<html lang="{{.Locale}}">
<head>
<meta charset="utf-8">
<script>
window.__i18n__ = {
  locale: "{{.Locale}}",
  messages: {{.Messages}},    // 已是 template.JS，安全注入
};
window.__t = function(key, args) {
  var tmpl = window.__i18n__.messages[key];
  if (typeof tmpl !== 'string') return '[' + key + ']';
  if (!args || typeof args !== 'object') return tmpl;
  return tmpl.replace(/\{(\w+)\}/g, function(m, name) {
    return (name in args) ? String(args[name]) : m;
  });
};
</script>
...
</head>
```

**POST /api/i18n/locale 验证 + 错误本地化**（NH5 + NM7 + NNM3）：

```
POST /api/i18n/locale
Content-Type: application/json
Cookie: nz_token=...

Body: {"locale": "en-US"}

验证（失败走 err := s.i18n.For(ResolveDashboard(...)).T(key, args)）：
  1. 复用 dashboard auth middleware（无 token → 401 纯英文 "unauthorized"）
  2. Content-Type 必须 application/json → 否则 400 + T("api.i18n.malformed_body")
  3. JSON 解码失败 / locale 字段缺失 → 400 + T("api.i18n.malformed_body")
  4. locale 长度 > 32 → 400 + T("api.i18n.invalid_locale", {locale: input})
  5. NormalizeLocale(locale) 不在 supported_locales → 400 + T("api.i18n.invalid_locale", {locale: input})
  6. 全部通过 → Set-Cookie + 200 + {"ok": true}

响应体格式：{"error": "...localized...", "code": "invalid_locale"}
```

**Cookie 属性**：

```
Set-Cookie: nz_lang=en-US; Path=/; Max-Age=31536000; SameSite=Lax
```

- **不** `HttpOnly`（前端可读以显示当前语言）
- **不** `Secure`（本地部署多；HTTPS 部署时 `cfg.Server.SecureCookie=true` 开启）
- `Max-Age=31536000`（1 年）
- `SameSite=Lax`

```go
type I18nConfig struct {

type HeuristicCfg struct {
    Enabled      bool    `yaml:"enabled"`
```

## 6. 风险与 Mitigation

| 风险 | 概率 | 影响 | Mitigation |

| `html/template` XSS | 低 | 高 | `template.JS` + `json.Marshal` + PR5a XSS 回归测试 |
| `/api/i18n/locale` CSRF | 低 | 低 | auth middleware + Content-Type + SameSite |
| session.Locale 持久化兼容 | 低 | 中 | 懒补幂等 + Go JSON 忽略未知字段（向前向后都兼容）|
| Bundle 并发 data race | 极低 | 高 | 不可变 + `-race` 覆盖 |
| embed.FS 不便 dev | 低 | 低 | `NAOZHI_DEV_LOCALE_DIR` + `-tags dev` + `make dev` |
| lint 误判 CJK 阻塞合流 | 中 | 低 | 基线文件 + 行级/同行/块级 ignore 注释（独立机制，非 build tag）|
| renderHTML 每次序列化字典 | 极低 | 低 | `FlatMessagesJS` 预序列化 + key 排序 + 字节稳定（可加 ETag）|
| `/lang auto` 未清除 → 永久锁定 | 已修复 | - | dispatcher 在 ResolveIM 之前短路清 session |
| Printer 持 map 引用阻碍 Reload | 已修复 | - | Printer 持 `*Bundle` 指针，未来 atomic.Pointer swap 可生效 |

## 8. 验收标准

### 功能
- [ ] 英文用户（启发式判定）在飞书私聊发 `/help`，收到英文回复
- [ ] 中文用户同一个 bot 发 `/help`，收到中文回复
- [ ] 飞书英文用户错判后，`/lang zh` 可纠正；后续消息稳定中文；`/lang auto` 可清除锁定
- [ ] Slack 用户 locale=en-US，发 `/help` 收到英文
- [ ] 跨 Slack workspace 同 user_id 缓存不串（回归测试覆盖）
- [ ] 单 Slack workspace 部署（team_id 空）不崩溃（NNM6）
- [ ] `/help` 输出包含 `/lang` 子命令说明（NNH3）
- [ ] `dashboard?lang=en` 可访问；reload 后仍英文（cookie 生效）
- [ ] `nz_lang=en-US` cookie 1 年持久化
- [ ] `POST /api/i18n/locale` 无 auth → 401；Content-Type 错 → 400 本地化文案；非白名单 locale → 400 本地化文案（NNM3）
- [ ] session state 文件无 Locale 字段 → 懒补成功，不影响首次 Save（幂等）

### 安全
- [ ] `html/template` XSS 测试：字典值含 `</script>`、`<img onerror>`、`';alert(1);//` 等能安全输出
- [ ] SameSite=Lax cookie 在跨站 POST 场景不发送
- [ ] `-tags dev` 未打开时 `NAOZHI_DEV_LOCALE_DIR` 被完全忽略

### 质量
- [ ] `go test ./internal/i18n/...` 全绿，覆盖率 >80%，带 `-race`
- [ ] CI lint 拦截 zh/en key 不对齐 / 占位符不对齐 / 非字面量 key / 基线外 CJK 新增
- [ ] 二进制尺寸 <60 KiB 新增
- [ ] Dispatcher 单消息延迟增量 <1 μs（benchmark）
- [ ] `renderDashboardHTML` 延迟增量 <500 μs（benchmark）
- [ ] `FlatMessagesJS` 多次调用返回字节相等（NNM10 稳定序）
